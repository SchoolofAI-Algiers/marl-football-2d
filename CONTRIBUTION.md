# Contribution Guidelines

Thank you for your interest in contributing to this project! To ensure a smooth and efficient workflow, please adhere to the following contribution guidelines.

## General Guidelines
- **Write Clean Code**: Maintain readability, simplicity, and efficiency.
- **Avoid Excessive AI-Generated Code**: Every part of the code must be justifiable, optimized, and structurally convenient.
- **Follow Project Standards**: Adhere to established coding conventions and best practices.

## Workflow
1. **Branching Strategy**
   - Always branch off from the `development` branch.
   - Use the following naming convention for your branches:
     ```
     [feature/fix/experiment]-[issue-number]-[contribution-name][contributor-name]
     ```
     Example:
     ```
     feature-15-environment-kamel
     ```
   
2. **Making Changes**
   - Ensure small, incremental updates.
   - Keep commits meaningful and well-documented.
   - Regularly pull changes from `development` to keep your branch updated.
   
3. **Opening a Pull Request (PR)**
   - Open a PR **from your branch into `development`** for each micro-update.
   - Provide a clear and descriptive PR title and description.
   - Ensure your code passes all tests and linting requirements.
   
4. **Merging to Main**
   - After completing a major task, the **project manager** will open a PR from `development` into `main`.

By following these guidelines, we can maintain a structured and efficient development process. Happy coding!
